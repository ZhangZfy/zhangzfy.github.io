Personal academic website for **Zhengfeiyang Finn Zhang**, deployed at `https://zhangzfy.github.io`.

Key identity fields in [`_config.yml`](_config.yml):

- `url: https://zhangzfy.github.io` + `baseurl:` (empty — personal site, not a project site)
- `scholar: last_name / first_name` — set to `[Zhang]` / `[Zhengfeiyang, Finn]`
- `_bibliography/papers.bib` — publications BibTeX file

## Essential Commands

```bash
# Start local dev server (site at http://localhost:8080)
docker compose pull && docker compose up

# Rebuild after Gemfile/Dockerfile changes
docker compose up --build

# Stop and free port 8080
docker compose down

# Format all files before committing (required — CI will fail without this)
npx prettier . --write
```

## Architecture

This is a **Jekyll 4.x static site** using the [al-folio](https://github.com/alshedivat/al-folio) theme.

Content lives in these directories:
| Directory | Purpose |
|---|---|
| `_pages/` | Static pages (about, cv, publications, projects, etc.) |
| `_posts/` | Blog posts — filename must be `YYYY-MM-DD-title.md` |
| `_projects/` | Project showcase entries |
| `_news/` | Announcements shown on the home page |
| `_teachings/` | Course/teaching pages |
| `_bibliography/papers.bib` | BibTeX publications |

Configuration and data:
| File | Purpose |
|---|---|
| `_config.yml` | Primary config — URL, features, scholar settings, plugins |
| `_data/socials.yml` | Social media links |
| `_data/cv.yml` | CV in RenderCV YAML format |
| `assets/json/resume.json` | CV in JSONResume format |
| `_sass/_themes.scss` | Theme color variables |

Build output goes to `_site/` (gitignored). The `gh-pages` branch is auto-generated by GitHub Actions — **never edit it directly**.

## Pre-commit Checklist

1. Run `npx prettier . --write` — CI enforces this via `.github/workflows/prettier.yml`
2. Test locally with `docker compose up` and visit `http://localhost:8080`
3. Check the terminal output for YAML/BibTeX parse errors before pushing

## Key Pitfalls

- **YAML special characters** — quote strings containing `:` or `#`: `title: "My: Site"`
- **`baseurl` must exist but be empty** for personal sites — do not delete the key
- **Scholar author name** in `_config.yml` must match BibTeX author spelling exactly for highlighting to work
- **Post filenames** must follow `YYYY-MM-DD-title.md` exactly or the post won't appear
- **ImageMagick required** for image processing — already installed in the Docker image
